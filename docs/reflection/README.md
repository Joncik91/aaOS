# Self-Reflection Log

A chronological record of aaOS reading its own code, finding bugs, proposing features, and having those results reviewed and sometimes shipped. Each entry is verified against git commits and observed behavior; where a number was estimated at the time and later corrected, both the original and the correction are recorded.

Each run or prep entry lives in its own file, dated `YYYY-MM-DD-<slug>.md`. New entries are added as new files, not appended to a monolith. For the build history that preceded this log, see [`../retrospective.md`](../retrospective.md). For cross-cutting lessons, see [`../patterns.md`](../patterns.md).

On cost figures: see [`cost-bookkeeping.md`](cost-bookkeeping.md) for the authoritative framing (dashboard > token math) and the running cumulative total.

- [`2026-04-17-self-build-tool-gap.md`](2026-04-17-self-build-tool-gap.md) — Runs 5 and 6 attempted the test-hygiene fix surfaced by run 4. Both failed to produce a diff, not from poor reasoning but from tool gaps: `file_read` dumps the whole file (no offset/limit), and there is no `file_edit` primitive — `file_write` rewrites the entire file, which for a 100KB source file with a 3-line change blows the 8KB output budget. Run 5 looped on re-reading server.rs trying to find a line number in a wall of text; run 6 fell silent trying to emit a 25k-token `file_write`. Fix applied by hand (`AAOS_ROLES_DIR` → nonexistent path + `ROLES_DIR_LOCK` mutex for env-var races). Three follow-ups queued: offset/limit on `file_read`, a new `file_edit` tool, and a recognition that the role's output budget is the wrong lever.

Empirically confirms the prior lesson: self-build is tool-bound, not model-bound.
