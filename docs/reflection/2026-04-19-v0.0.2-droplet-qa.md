# 2026-04-19 — v0.0.2 droplet QA (verification pass)

Re-ran the five-pass QA suite against the shipped v0.0.2 `.deb` on the same fresh Debian 13 droplet as the v0.0.1 run, to confirm Bugs 1–6 are actually closed end-to-end.

## Setup

- Same droplet as v0.0.1 QA (Debian 13, kernel 6.12.43, Landlock active in LSM stack).
- `dpkg --purge aaos` → `apt install ./aaos_0.0.2-1_amd64.deb`. Clean slate per pass.
- Same DeepSeek key in `/etc/default/aaos` as the v0.0.1 run (mode 0600 root:root).

## Pass 1 — Package hygiene

- **Lintian: 0 errors, 0 warnings.** (v0.0.1: 11 errors + 9 warnings.)
- 80 files in the archive: systemd unit under `usr/lib/systemd/`, `copyright` referencing common-licenses, `changelog.Debian.gz`, `lintian/overrides/aaos`, `mcp-servers.yaml.example` (3098 bytes), 21 skills under `/usr/share/aaos/skills/`, 4 roles.
- Deps lean: `libc6, libssl3t64, systemd, ripgrep, ca-certificates`.
- Installed-Size 14791 KB (v0.0.1: 18162 KB). Stripping saved ~3.4 MB.

## Pass 2 — Daemon lifecycle

- `systemctl is-active agentd` → `active`. `is-enabled` → `enabled`.
- Socket: `/run/agentd/agentd.sock` mode 0660 owned `aaos:aaos`.
- User + group created by `postinst`. Purge removes both.
- Startup log sequence: `starting agentd` → `DeepSeek LLM client configured` → `using in-memory memory store` → `using NamespacedBackend` → `MCP disabled (copy .example to enable)` → `workspace-gc: startup sweep complete` → `listening on unix socket`.
- **Bug 5 closed**: `using NamespacedBackend` count per restart = 1 (v0.0.1: 2).
- **Bug 4 closed**: MCP-off state explicitly logged (`no /etc/aaos/mcp-servers.yaml — MCP disabled (copy /etc/aaos/mcp-servers.yaml.example to enable)`).

## Pass 3 — Canonical goal

- `agentd submit "fetch https://news.ycombinator.com and write the top 5 story titles to /data/hn.md"` — **27 s** wall clock (v0.0.1 first run: 57 s; v0.0.2 immediately post-rebuild: 41 s).
- Output file: 2025 bytes, 5 stories with author + points + comment counts. Real current content — "Vercel April 2026 Security Incident" at 332 points (was 280 in earlier run; genuinely live).
- **Bug 1 closed**: writer agent's `file_read` + `file_write` audit-tagged `[worker]`; fetcher (scaffold) `[daemon]` as designed.
- **Bug 2 closed**: `ps -eo stat | grep -c "^Z"` = 0 after run (v0.0.1: 1 zombie per run accumulating).

## Pass 4 — Fault injection

- **Bad API key**: CLI now prints `error: terminal: ipc error: planner LLM: authentication failed — check API key` + journalctl pointer hint; daemon emits structured `ERROR agentd::server: plan execution failed error=... run_id=<uuid>`. **Bug 3 closed.** (v0.0.1: silent `bootstrap failed (0k in / 0k out, 0s)` with zero log lines.)
- **Corrupt role YAML** (garbage overwrite on `writer.yaml`): `ERROR agentd::server: role YAML failed to parse — this role will not load. Fix the file or remove it, then restart agentd. file=/etc/aaos/roles/writer.yaml error=... at line 1 column 14` + `ERROR agentd::server: role catalog FAILED TO LOAD; computed orchestration disabled. See per-file errors above.`  Exact file, line, column, remediation. Unchanged from v0.0.1 (already correct).
- **Unreachable MCP server** (http://127.0.0.1:9999): daemon logs `INFO agentd: MCP client: attempting to connect to 1 configured server(s) servers=1` then `INFO agentd: MCP client: 0 of 1 server(s) registered (others retrying in background) registered=0 attempted=1`. Daemon stays up, socket still serving. **Bug 4 closed.** (v0.0.1: zero MCP log lines at all.)

## Pass 5 — MCP + skills

- **MCP server**: `server.enabled: true` in config → startup log `MCP server: starting loopback listener bind=127.0.0.1:3781`. `ss -tlnp` confirms listener. `tools/list` over HTTP returns the three tools (`submit_goal`, `get_agent_status`, `cancel_agent`).
- **Skills catalog**: 21 skills under `/usr/share/aaos/skills/` as expected. `context-engineering/SKILL.md` parses cleanly with valid frontmatter.
